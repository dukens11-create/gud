# App Store Screenshot Guide

Guidelines for preparing screenshots for Google Play Store and Apple App Store submissions.

## ğŸ“± Google Play Store Requirements

### Screenshot Specifications
- **Format**: PNG or JPEG
- **Minimum Dimension**: 320px
- **Maximum Dimension**: 3840px
- **Required**: At least 2 screenshots
- **Maximum**: 8 screenshots per type
- **Aspect Ratio**: 16:9 recommended

### Recommended Sizes
- Phone: 1080 x 1920 (portrait) or 1920 x 1080 (landscape)
- 7-inch Tablet: 1200 x 1920
- 10-inch Tablet: 1600 x 2560

### What to Include
1. Login/Welcome screen
2. Main dashboard (both driver and admin)
3. Key feature screens (load management, POD upload)
4. Statistics/analytics view
5. Any unique selling points

### Before Submission

- [ ] All screenshots are correct size/resolution
- [ ] No personal data visible
- [ ] App looks professional and polished
- [ ] Key features are highlighted
- [ ] Screenshots tell a story/flow
- [ ] Text is readable

### Google Play Store Specific
- [ ] At least 2 screenshots uploaded
- [ ] Feature graphic prepared (1024 x 500)
- [ ] App icon prepared (512 x 512)

### Apple App Store Specific
- [ ] Screenshots for required device sizes
- [ ] App Preview video (optional, recommended)
- [ ] Screenshots ordered properly
