# Screenshot Requirements for GUD Express

**Last Updated:** February 6, 2026  
**Version:** 1.0

This document outlines all screenshot requirements for both Google Play Store and Apple App Store submissions.

---

## Overview

Screenshots are essential for converting app store visitors into users. They must:
- **Showcase key features** clearly and effectively
- **Tell a story** about how the app works
- **Be high quality** and professional
- **Follow platform guidelines** precisely
- **Represent actual app content** accurately

---

## Google Play Store Screenshot Requirements

### Technical Specifications

**Phone Screenshots:**
- **Minimum:** 320 pixels on shortest side
- **Maximum:** 3840 pixels on longest side
- **Required:** At least 2 screenshots
- **Maximum:** 8 screenshots
- **Aspect Ratio:** 16:9 or 9:16 recommended
- **Format:** PNG or JPEG (PNG recommended for quality)

**Common Phone Sizes:**
- 1080×1920 (Full HD, 9:16 portrait)
- 1440×2560 (QHD, 9:16 portrait)
- 1920×1080 (Full HD, 16:9 landscape)

**Recommended for GUD Express:** 1080×1920 (portrait)

**7-inch Tablet Screenshots (Optional):**
- **Size:** 1024×1768 or 1768×1024
- **Required:** At least 2 if provided
- **Maximum:** 8 screenshots
- **Note:** Optional but recommended for tablet users

**10-inch Tablet Screenshots (Optional):**
- **Size:** 1200×1920 or 1920×1200
- **Required:** At least 2 if provided
- **Maximum:** 8 screenshots
- **Note:** Optional but recommended for tablet users

### Design Guidelines

**Content Requirements:**
- Must show actual app content (no mockups)
- Can include device frames (optional)
- Can include text captions/annotations
- Should highlight key features
- Must be clear and readable

**What to Avoid:**
- Blurry or low-resolution images
- Misleading or exaggerated features
- Content that violates policies
- Excessive text that obscures the UI
- Copyrighted or trademarked content

**Best Practices:**
- Use consistent design across all screenshots
- Show progression or user flow
- Include captions to explain features
- Use actual data (or realistic test data)
- Ensure text is readable on mobile devices
- Test how they appear in Play Store

### Screenshot Captions

Google Play allows optional text overlays on screenshots:
- Keep captions short and impactful
- Use large, readable fonts (minimum 36px)
- Ensure good contrast with background
- Highlight key benefits, not just features
- Use consistent caption style

## Apple App Store Screenshot Requirements

### Technical Specifications

Apple requires screenshots for specific device sizes. You must provide at least one screenshot for each device type you support.

**iPhone 6.7" Display (Required)**
- **Devices:** iPhone 14 Pro Max, iPhone 15 Pro Max, iPhone 15 Plus
- **Size:** 1290×2796 pixels (portrait) or 2796×1290 (landscape)
- **Alternative:** 1284×2778 or 2778×1284
- **Required:** Minimum 1, Maximum 10 screenshots
- **Priority:** HIGH (required for submission)

**iPhone 6.5" Display (Required)**
- **Devices:** iPhone 11 Pro Max, iPhone XS Max, iPhone 11, iPhone XR
- **Size:** 1242×2688 pixels (portrait) or 2688×1242 (landscape)
- **Alternative:** 1284×2778 or 2778×1284
- **Required:** Minimum 1, Maximum 10 screenshots
- **Priority:** HIGH (required for submission)

**iPhone 5.5" Display (Optional)**
- **Devices:** iPhone 8 Plus, iPhone 7 Plus, iPhone 6s Plus
- **Size:** 1242×2208 pixels (portrait) or 2208×1242 (landscape)
- **Required:** Minimum 1, Maximum 10 screenshots
- **Priority:** LOW (legacy devices, optional)

**iPad Pro (12.9-inch) 2nd Gen (Recommended)**
- **Size:** 2048×2732 pixels (portrait) or 2732×2048 (landscape)
- **Required:** Minimum 1, Maximum 10 screenshots
- **Priority:** MEDIUM (if supporting iPad)

**iPad Pro (11-inch) (Optional)**
- **Size:** 1668×2388 pixels (portrait) or 2388×1668 (landscape)
- **Required:** Minimum 1, Maximum 10 screenshots
- **Priority:** LOW (optional)

### Design Guidelines

**Content Requirements:**
- Must show actual app interface
- Must accurately represent the app
- Can include text overlays and graphics
- Can use device frames (but not required)
- Must be actual screenshots or captured video frames

**What Apple Prohibits:**
- Simulated screenshots or mockups (must be actual)
- Screenshots that don't represent current version
- Misleading features or functionality
- Inappropriate or offensive content
- Screenshots from other apps or devices

**Best Practices:**
- Show the app's best features first
- Use high-quality captures
- Add descriptive text to highlight features
- Maintain consistent visual style
- Ensure all text is legible
- Show user flows or workflows
- Include localized screenshots for each language

## Screenshot Creation Process

### Step 1: Prepare Test Data

Create realistic test data that represents actual use:
- **Loads:** Create 10-15 sample loads with varied statuses
- **Drivers:** Add 5-8 test drivers
- **PODs:** Upload professional-looking delivery photos
- **Earnings:** Generate realistic earnings data
- **Dates:** Use recent dates for current feel

**Test Data Guidelines:**
- Use professional-sounding company names
- Use realistic addresses (real cities, generic street names)
- Use reasonable dollar amounts
- Avoid any real personal information
- Ensure diversity in data (statuses, locations, etc.)

### Step 2: Capture Screenshots

**On Android:**

Method 1: Physical Device
- Set up test account with sample data
- Navigate to each screen
- Press Power + Volume Down simultaneously
- Screenshots saved to Pictures/Screenshots

Method 2: Android Emulator
- Use Android Studio AVD Manager
- Configure device to required resolution
- Use emulator controls to capture
- Or use Ctrl+S (Cmd+S on Mac)

**On iOS:**

Method 1: Physical Device
- Set up test account with sample data
- Navigate to each screen
- Press Side Button + Volume Up
- Screenshots saved to Photos app

Method 2: iOS Simulator
- Use Xcode Simulator
- Configure device to required size
- Cmd+S to save screenshot
- Or use simulator screenshot option

### Step 3: Edit and Enhance

**Tools:**
- **Photoshop:** Professional editing
- **Figma:** Design and annotate
- **Sketch:** Mac-only design tool
- **Canva:** Quick online editing
- **Previewer (online):** Add device frames

**Editing Steps:**
1. **Crop** to exact required dimensions
2. **Add device frame** (optional but recommended)
3. **Add text overlays** with feature descriptions
4. **Ensure consistency** across all screenshots
5. **Optimize file size** without quality loss
6. **Export** in correct format (PNG recommended)

**Text Overlay Guidelines:**
- **Font:** Sans-serif, bold, 48-72px
- **Color:** High contrast with background
- **Position:** Top or bottom, not covering critical UI
- **Shadow:** Add drop shadow for readability
- **Style:** Consistent across all screenshots

### Step 4: Quality Check

Before uploading, verify:
- [ ] Correct dimensions for each platform
- [ ] High resolution and sharp
- [ ] No placeholder text (unless intentional)
- [ ] No personal information visible
- [ ] Professional appearance
- [ ] Captions are readable on mobile
- [ ] Consistent branding and style
- [ ] File sizes reasonable (under 5 MB each)
- [ ] File names clear and organized

## Screenshot Upload Process

### Google Play Store

1. Log in to Google Play Console
2. Navigate to App > Store presence > Main store listing
3. Scroll to "Phone screenshots" section
4. Click "Add phones screenshots"
5. Upload 2-8 screenshots (drag and drop or browse)
6. Reorder by dragging (first shown first in store)
7. Save changes
8. Preview in Play Store

**Tips:**
- First 2-3 screenshots are most important
- Users see first 3 on search results
- Reorder anytime without new review

### Apple App Store

1. Log in to App Store Connect
2. Select your app
3. Go to App Store tab
4. Scroll to "App Preview and Screenshots"
5. Select device size
6. Upload screenshots (minimum 1, maximum 10)
7. Reorder by dragging
8. Add screenshots for each required device size
9. Save and submit with app

**Tips:**
- Must provide for all required device sizes
- First screenshot appears in search results
- Can localize for different languages
- Screenshots reviewed with app submission
